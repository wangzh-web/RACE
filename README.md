# RACE: Risk-Aware Cloud-Edge Orchestration with Online Adaptive Calibration for Trustworthy LLM Services

[![Python 3.10+](https://img.shields.io/badge/python-3.10+-blue.svg)](https://www.python.org/downloads/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

Reference implementation, trained SEP checkpoint, and a **deterministic replay bundle** for RACE,
a framework for trustworthy edge-cloud orchestration of large language model services.

> RACE reformulates edge-cloud LLM scheduling as **risk allocation**: the user sets a maximum
> acceptable hallucination rate α, and RACE dynamically calibrates a routing threshold — via a
> lightweight **Semantic Entropy Probe (SEP)** and **Projected Online Gradient Descent (P-OGD)** —
> to minimize the **Cloud Invocation Rate (CIR)** subject to the risk constraint, with provable
> **bounded (T-independent) regret** and finite-sample coverage guarantees.

**Cloud worker:** GPT-5 (`gpt-5-2025-08-07`, reasoning `low`). **Edge models:** Phi-3-medium, Mistral-7B.

---

## Reproducibility at a glance

Every reported number is regenerated by a **deterministic replay** over recorded per-sample edge
outputs and a cached temperature-0 GPT-5 answer set — **no GPU or API key is needed to reproduce
the results**. The replay is the single source of truth for all CIR, EdgeErr, and ablation values.

```bash
# 0. Environment (pinned)
pip install -r requirements-frozen.txt

# 1. Main comparison — all baselines + RACE, both datasets
python replay/replay_baselines.py --dataset mmlu
python replay/replay_baselines.py --dataset triviaqa

# 2. RACE's canonical operating point (Algorithm 1)
python replay/replay_compare.py \
    --records data_records/persample_phi3_mmlu_test.jsonl --score-key u_sep_mmlu --alpha 0.30
#   -> MMLU: CIR = 6.25%, EdgeErr = 29.33%   (TriviaQA: 4.05% / 28.09%)

# 3. Ablation suite (α-sweep, ρ-sweep, envelope factor grid, online-vs-static)
python replay/run_ablation_replays.py \
    --records data_records/persample_phi3_mmlu_test.jsonl \
    --cal     data_records/persample_phi3_mmlu_cal.jsonl \
    --score-key u_sep_mmlu --out replay/out_mmlu
```

`replay/replay_baselines.py` treats CIR/EdgeErr as a validation gate: they reproduce the reported
values exactly (system accuracy additionally folds in the cached GPT-5 answers).

> **Two RACE implementations.** `replay/replay_compare.py` runs both the canonical Algorithm 1
> (sliding-window P-OGD; CIR = 6.25% on MMLU, the reported value) and a legacy per-sample-update
> variant (CIR = 8.15%) side by side over identical inputs, so the difference is fully transparent.

## Models and datasets

| Role | Hugging Face ID |
|---|---|
| Edge (primary) | `microsoft/Phi-3-medium-4k-instruct` |
| Edge (secondary) | `mistralai/Mistral-7B-Instruct-v0.3` |
| Benchmark | `cais/mmlu` |
| Benchmark | `mandarjoshi/trivia_qa` |

Dependency versions (including `transformers` and `datasets`, which fix the default model/dataset
revisions) are pinned in [`requirements-frozen.txt`](requirements-frozen.txt). Calibration uses a
500-sample split disjoint from the 2,000-sample test stream of each benchmark.

**Random seeds:** `{42, 123, 456, 789, 1024, 2048, 3141, 4096, 5555, 6789}`
(used for stochastic baselines and the shuffled-stream study; the canonical RACE replay is
deterministic and seed-independent).

## Notes

- The cached GPT-5 answers (`replay/cloud_gpt5_*.jsonl`) and per-sample edge records
  (`data_records/*.jsonl`) contain only model responses, correctness flags, token counts, and
  latencies — **no API keys or credentials**. Scripts that call the cloud API read `OPENAI_API_KEY`
  from the environment; none is stored in this repository.
- To regenerate records from scratch (GPU required), use `src/generate_sep_data.py` and
  `src/train_sep*.py`; the released `data_records/` and `checkpoints/phase2_sep_model.pt` let you
  skip that step and reproduce all results directly.
